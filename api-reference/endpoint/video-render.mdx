## Endpoint

```
POST https://api.ugc.inc/video/render
```

## Overview

Create a custom video by composing multiple layers of video clips, images, text, and audio. The Editor structure provides pixel-perfect control over timing, positioning, effects, and transitions.

**Use cases:**
- Generate automated video content at scale
- Add dynamic text overlays to video templates
- Compose multi-layer video effects
- Create slideshow-style videos with custom timing
- Build video editors within your application

## Request Body

<ParamField body="editor" type="Editor" required>
  Video composition configuration
  
  See the [Editor data structure](/data-structures/editor) for complete details.
  
  <Expandable title="Editor structure">
    <ParamField body="editor.width" type="number" required>
      Output video width in pixels (e.g., 1080)
    </ParamField>
    
    <ParamField body="editor.height" type="number" required>
      Output video height in pixels (e.g., 1920)
    </ParamField>
    
    <ParamField body="editor.fps" type="number" required>
      Frames per second (typically 30)
    </ParamField>
    
    <ParamField body="editor.duration" type="number">
      Total duration in milliseconds
      
      If omitted, calculated automatically from segments
    </ParamField>
    
    <ParamField body="editor.channels" type="Channel[]" required>
      Array of channels containing segments
      
      Each channel represents a layer in the composition
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="data" type="object">
  <ResponseField name="video_url" type="string">
    Public URL to the rendered video file
    
    The video is ready for download or posting
    
    Format: MP4 with H.264 video and AAC audio
  </ResponseField>
</ResponseField>

## Processing Time

Video rendering is CPU-intensive and processing time varies based on:
- **Duration:** Longer videos take more time
- **Complexity:** Number of segments and effects
- **Resolution:** Higher resolution requires more processing

**Typical processing times:**
- 5 second video: 10-30 seconds
- 15 second video: 30-90 seconds
- 30 second video: 60-180 seconds

<Tip>
For production use, consider rendering videos asynchronously and polling for completion status.
</Tip>

## Segment Types

The editor supports multiple segment types:

| Type | Description | Example Use |
|------|-------------|-------------|
| `video` | Video clip with playback control | Background footage, clips |
| `image` | Static or animated images | Logos, GIFs, overlays |
| `text` | Rich text with full typography | Captions, titles, subtitles |
| `audio` | Background audio or music | Voiceover, soundtrack |
| `editor` | Nested composition | Reusable templates, effects |

## Advanced Features

### Nested Compositions

Use `editor` segments to create reusable templates:

```json
{
  "type": "editor",
  "editor": {
    "width": 1080,
    "height": 300,
    "fps": 30,
    "channels": [/* nested composition */]
  }
}
```

### Time-based Animations

Combine `offset` and `duration` for precise timing:

```json
{
  "offset": { "type": "absolute", "value": 1000 },
  "duration": { "type": "absolute", "value": 2000 }
}
```

### Multi-layer Effects

Stack segments with different `zIndex` values:
- Background video: `zIndex: 0`
- Image overlay: `zIndex: 5`
- Text caption: `zIndex: 10`

<Warning>
**Performance Considerations:**
- Rendering is CPU-intensive - expect 2-4x processing time vs video duration
- Keep segment count reasonable (< 50 per composition)
- Use optimized source media (compressed videos, appropriate resolutions)
- For high-volume use, implement queuing and rate limiting
</Warning>
